# FlixGames Kick-off — 2026-07-14

Participants: Franz von Esebeck, Emma Sporn, David Cisar, Daniel Leon Glauert, Carolina Fromm

## Summary

Focus: process standardization through automated content creation, and strategic planning for the hackathon.

- **Referent tools / production automation** — new input mask for referents ("Referenten-Eingabemaske") to define their case study (Kasuistik) up front, with a live preview. Goal: eliminate manual slide design and raise quality via templates. Referents place placeholders for camera shots and graphics directly, so production is wired straight into the slides — speak once, get a finished video. Two modes agreed: build from scratch on the standardized system, or AI-assisted conversion of existing PowerPoint slides (extract bullet points, pick a matching template, pre-fill, plus spellcheck/compliance checks).
- **Template strategy** — a limited set (~8 variants) covering the range of referent content, removing the layout burden from referents.
- **Technical approach** — export as PowerPoint to keep the current production pipeline (XML-based placeholder control) rather than rebuilding it from scratch.
- **Pharma Space / Pharma dashboard** — central pharma area on the site: customers can view topics, configure offers, book projects, and see pipeline status. Goal: automate and add transparency to the whole sales process. Pharma handover forms feed standardized data straight into the project database to improve expectation management.
- **Hackathon scope** — theme locked in as **process standardization + automated content/video creation**. Prototype should simulate the pipeline: user picks a topic from a list, uploads content, and a video gets generated from it. Full FCP video-editor integration is out of scope for the hackathon — keep focus on the core demo.
- **Pitch/ROI** — pitch needs a clear story plus a time-savings estimate (ROI), framed as an ambitious innovation project.
- **Mockup demo** — Daniel showed a Keynote mockup of a live production without static slides (full-screen presenter with dynamic overlays); non-destructive editing (tracks stay editable afterward) is a hard requirement even in mockup mode.
